Scope: Finish the Admin dashboard so that every metric, table, and chart pulls real-time data from production tables and Stripe webhooks.
Strict rule: no hard-coded sample rows or dummy JSON. All counts and dollar totals must reflect actual users, teams, credits, and payments.

1  Data Sources & Wiring

Module	Source	Endpoint / Query
Users	users table	SELECT id,email,plan,credit_balance,last_login FROM users ORDER BY created_at DESC
Teams	teams, team_members tables	JOIN to list orgs, member counts, usage
Credits	credit_ledger	Aggregate SUM(credits_used) per user / team
Affiliates	affiliates, referrals	Show sign-ups, conversions, payout status
JV Deals	jv_deal_submissions	New → Pending → Approved / Denied
Billing	Stripe Webhooks (customer.subscription.*, invoice.payment_succeeded)	Mirror to billing_events table; query totals & MRR

Requirement: Every widget must query one of these canonical tables; no inline arrays allowed.
