Apply a consistent “$1,000,000” style (U S locale, comma-separated thousands, no cents) to every UI field that represents a price, cost, or monetary value—throughout the entire app—without touching existing calculations, layouts, or logic in the Acquisitions, Dispositions, Comps, or Construction modules.

1. Scope & Rules
Area	Action
All price inputs (text fields, sliders, tables)	• Display as $1,234,567
• Strip non-numeric characters on focus → user types digits only
• Re-format on blur
All price outputs (labels, summary cards, charts)	Format with Intl.NumberFormat('en-US', {style:'currency', currency:'USD', minimumFractionDigits:0})
Excel-driven calculators (Fix-&-Flip, New-Build)	Parse raw numbers → store/read as integers; render formatted string only in view layer
PDF/CSV exports & copies	Preserve same $1,234,567 style
Backend math & API payloads	Accept/return plain integers (no $, no commas) — ensure a clean parse step before calculations

No cents at any time (we round to whole dollars).

// utils/formatCurrency.ts
export const formatUSD = (n: number) =>
  new Intl.NumberFormat('en-US', {
    style: 'currency',
    currency: 'USD',
    minimumFractionDigits: 0,
    maximumFractionDigits: 0,
  }).format(n);

const InputCurrency = ({value, onChange, ...rest}) => {
  const parse = (v:string) => Number(v.replace(/[^0-9]/g, ''));
  return (
    <input
      {...rest}
      value={formatUSD(value)}
      onFocus={e=>e.target.select()}
      onChange={e=>onChange(parse(e.target.value))}
      onBlur={e=>onChange(parse(e.target.value))}  // re-format via parent state update
    />
  );
};

Edge-case handling

Empty → show blank, not “$0”.

Negative values (refunds) → “–$1,000” (use signDisplay:'exceptZero').

Disable formatting in spreadsheet-style CSV import dialogs until confirmation step.
