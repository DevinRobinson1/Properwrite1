Why it’s still blank
The helper isn’t being called

The log line “Submitting with Google Autocomplete (New) data” still shows the raw, duplicated address.

That means the new to_zillow_search_string() function was added but not wired in before the API call.

No HTTP request is leaving the browser
Open the Network tab ⇒ filter by propertyExtendedSearch. If you see zero rows, the fetch is never triggered (likely because the backend returned early with INCOMPLETE_ADDRESS).

Backend can’t see the RapidAPI credentials in prod
A 400 from your own /property-analysis route combined with ERR_BLOCKED_BY_CLIENT for the RapidAPI URL means the server-side call never happened—often an empty process.env.RAPIDAPI_KEY.
