🛠 Replit Debug Prompt – Zillow/Redfin/Realtor data not loading
Context

On every address we test the “Property Valuations” cards come back “Not available (Tried: Zillow/Redfin)” and the Network pane shows our backend returning 400 INCOMPLETE_ADDRESS.

Console reveals the Autocomplete handler is called repeatedly, then we POST to /property-analysis without a fully-parsed address payload (street, city, state, zip or lat/lng).

Result: upstream scrapers / data vendors short-circuit, so nothing reaches the UI.

1 Root cause we can see
Evidence	Likely reason
Request body logged in dev-tools is missing street or lat,lng.	We’re grabbing only suggestions[0].placePrediction.text.text from the Autocomplete response instead of the place ID.
Zillow & Redfin helpers expect either (street, city, state, zip) or GPS coords.	We never geocode the prediction after Autocomplete, so scrapers get partial input.

2 Fix we need (no UI changes!)
Autocomplete → PlaceID → Place Details → formatted address

Keep the user text field exactly as-is.

Use Autocomplete (New) only to fetch the best prediction and its placeId.

Immediately call Place Details (New) with that placeId and X-Goog-FieldMask: formattedAddress,location.

Split the returned formattedAddress into street, city, state, zip (or just pass the whole string plus lat/lng).

Update submitProperty() (or analyzeProperty() backend route)

If any of street && city && state is missing, fallback to geocode and fill.

Throw a single, user-friendly toast (“Couldn’t validate that address, please refine.”) instead of the JS alert.

Keep processing pipeline identical once a complete address object is available—this preserves all Acquisition & Disposition UI logic.

Important: Do not rename or move any of the Acquisition/Disposition sections—just patch the data-fetch layer.

// in /services/google.ts
export async function resolveAddressViaPlaces(input: string, sessionToken: string) {
  const acResp = await fetch(
    'https://places.googleapis.com/v1/places:autocomplete',
    { method:'POST',
      headers: { 
        'Content-Type':'application/json',
        'X-Goog-Api-Key': process.env.GMAPS_KEY,
        'X-Goog-FieldMask':'suggestions.placePrediction.placeId'
      },
      body: JSON.stringify({ input, sessionToken })
    });
  const placeId = (await acResp.json())
      ?.suggestions?.[0]?.placePrediction?.placeId;
  if (!placeId) throw new Error('NO_PREDICTION');
  
  const details = await fetch(
    `https://places.googleapis.com/v1/places/${placeId}`,
    { headers:{
        'X-Goog-Api-Key': process.env.GMAPS_KEY,
        'X-Goog-FieldMask':'formattedAddress,location'
      }});
  return await details.json(); // contains formattedAddress + lat/lng
}
