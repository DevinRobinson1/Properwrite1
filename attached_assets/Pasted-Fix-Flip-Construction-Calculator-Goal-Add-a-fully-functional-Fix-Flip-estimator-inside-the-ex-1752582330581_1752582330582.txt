Fix & Flip Construction Calculator

Goal
Add a fully-functional “Fix & Flip” estimator inside the existing Construction tab (sibling to New Build). The estimator must read unit-costs from the Excel price sheet the user just uploaded and let users edit those costs in-app without touching any other sections (Acquisitions, Dispositions, Comps, etc.).

3 . Dynamic Price Sheet
Create a reusable usePriceSheet() hook that:

Loads default costs from priceSheet table (PostgreSQL) seeded from Copy of Full Renovation Estimator (Installer).xlsx “Price Sheet” tab.

Allows Org Admins to edit any unit price in a Price Sheet Modal (<PriceSheetEditor/>).

Persists edits via Supabase RPC; caches in Zustand for fast reads across the estimator.

Emits priceSheetUpdated event so open estimators recalculate instantly.

UI pattern:
• “Price Sheet ⚙️” pill in the estimator’s top-right.
• Opens a full-screen modal with a spreadsheet-like table (ag-Grid, editable).
• Columns: Category – Item – Unit – Default Cost – Org Override Cost.
• “Revert to Default” & “Save” CTA.

No hard-coded numbers in components – always call the hook.

4 . Excel → DB Seeder
Add a one-time scripts/seedPriceSheet.ts to parse both uploaded Excel files:

Sheet “Price Sheet”: for Fix & Flip.

Sheet “New Construction Breakdown – Costs”: already used in the New-Build calculator.

Map columns → (category, item, unit, default_cost_cents).

On deploy / npm run seed, populate priceSheet table if empty.

Use xlsx npm package (already in project) – do NOT bundle Excel files in client build.

5 . State, Validation & Credits
Each estimator page consumes 1 credit on first Save. Re-opens of same deal are free.

Add useCredits() guard – if credits exhausted show existing “Upgrade / Buy Pack / Pay-with-Bitcoin” modal.

Validate:

Quantity ≥ 0 and sensible (e.g., 0–10 000).

Required fields (ARV, Total SF) must be numbers.

Summary total >= $0 before enable “Save”.
