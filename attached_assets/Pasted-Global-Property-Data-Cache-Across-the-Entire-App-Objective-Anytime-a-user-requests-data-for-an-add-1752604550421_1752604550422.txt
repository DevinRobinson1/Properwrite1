Global Property-Data Cache (Across the Entire App)
Objective
Anytime a user requests data for an address we’ve already fetched anywhere in the platform—Acquisitions, Dispositions, Comps, or Construction—the app should first look in a shared cache and reuse the saved payload instead of hitting Zillow, Redfin, RentCast, Google, etc. again.
• Do not change layouts or business logic—just insert a smart cache layer.
• Must work org-wide (all teammates share the same cache).

1. Cache Design
Item	Spec
Key	sha256(normalized_full_address) and any unique IDs we discover (e.g., zpid, place_id).
Store Priority	1️⃣ Redis (⟨10 ms). 2️⃣ Fallback: Supabase table property_cache.
Payload	Unified object:
{ address, zpid, place_id, comps, arv, rentZestimate, bed, bath, sqft, photos, fetched_at }
TTL	24 h default; extend to 72 h if the API quota is tight (configurable in .env).
Hit / Miss Flow	1. getPropertyData(address) wrapper checks cache.
2. If hit + TTL valid → return payload.
3. If miss or stale → hit external APIs → store new payload.

export const normalizeAddress = (a:string)=>
  a.trim().toLowerCase().replace(/\s+/g,' ');
export const keyFor = (addr:string)=> 
  crypto.createHash('sha256').update(normalizeAddress(addr)).digest('hex');

// services/getPropertyData.ts
export async function getPropertyData(fullAddress:string){
  const key = keyFor(fullAddress);
  const cached = await cache.get(key);            // redis or supabase
  if (cached && Date.now()-cached.fetched_at < TTL_MS){
     return cached.payload;
  }
  const fresh = await fetchAllApis(fullAddress);   // existing logic
  await cache.set(key,{ payload:fresh, fetched_at:Date.now()});
  return fresh;
}
Replace direct API calls (Zillow, Redfin, RentCast, Google Places) in every feature module with this wrapper.

  fetched_at timestamptz default now()
);
create index on property_cache (fetched_at);
Concurrency guard

Use Redis SETNX or Supabase advisory lock so simultaneous users hitting the same unseen address don’t trigger parallel external calls.*

Stale-while-revalidate

On cache hit that’s older than TTL but < 7 days, return cached data immediately, then re-fetch in background and update cache (improves UX).*

3. Testing & Validation
Test	Expectation
Cold search	First query logs external API hits, stores to cache.
Repeat search (≤24 h)	No external calls; returns in <200 ms.
Load from different tab (Dispositions → Comps)	Uses same cached payload.
TTL expired	New API call, cache refreshed.
Concurrency	Two users searching same new address triggers exactly one external call.
Offline API	If cache available, show cached data + “Using saved info (API offline)” banner.

Unit-test with Redis mocked; Cypress end-to-end verifying time difference between first and second fetch.
