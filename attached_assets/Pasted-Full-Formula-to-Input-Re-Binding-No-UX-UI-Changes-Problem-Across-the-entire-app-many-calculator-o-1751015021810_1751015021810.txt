Full Formula-to-Input Re-Binding (No UX/UI Changes)
Problem Across the entire app, many calculator outputs stay frozen after a user changes an input box (e.g., Repairs, ARV, Commissions).
Goal Without touching styling or markup, verify that every numeric input is wired to a single source of truth and that all formulas recompute on change.

1. Build a Master Map of Inputs ↔ State ↔ Formulas
Create formulaBindingMap.ts (or JSON) listing, for each page & strategy, the triple:

Page / Tab	Input Field (DOM id or label)	State Key	Down-stream Formulas
Acq → Wholesale	#repairs-input	deal.repairs	wholesaleMAO, Ackerman tiers
Acq → Installment	#listingDiscount	novation.discount	novationMAO, Net-to-Seller
Dispo → Cash Sale	#wholesalePct	wholesale.pct	Cash-Buyer card prices
…	…	…	…

Populate the table for all variable boxes you find.

2. One Source of Truth
All inputs must read and write to Context/Redux values only—no local duplicates.

Example pattern:

jsx
Copy
Edit
const { repairs } = useDealState();
<input
  value={formatCurrency(repairs)}
  onChange={e => dispatch(setRepairs(cleanNumber(e.target.value)))}
/>
3. Reactive Hooks / useMemo
For every calculator hook:

tsx
Copy
Edit
const { arv, repairs, pct, profit } = useDealState();
return useMemo(() => {
  const mao = arv * pct - repairs - profit;
  return { mao, ack: [mao*0.65, mao*0.85, mao*0.95, mao] };
}, [arv, repairs, pct, profit]);
List all referenced keys in the dependency array so recalculation fires immediately.

5. Manual QA Checklist
Wholesale MAO: change ARV, Repairs, Assignment Fee → MAO & Ackerman update.

Installment / Novation: edit Closing Fees %, Min-Profit → MAO & Net to Us update.

Subject-To: change Cash-to-Seller, Arrears, PITI → Cash-to-Close + Cash-Flow update.

Seller Finance: adjust Down-Payment slider, Wrap Term → Price, PI, Cash-Flow update.

Dispositions Compare All: reflect new numbers after any edit.

No visible layout shifts should occur.
