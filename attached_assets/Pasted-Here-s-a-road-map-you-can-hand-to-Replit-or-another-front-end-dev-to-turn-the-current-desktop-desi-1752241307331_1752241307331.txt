Here’s a road-map you can hand to Replit (or another front-end dev) to turn the current desktop design into a slick, phone-first experience without touching the Acquisitions / Dispositions logic.

1. Layout strategy 📐
Area	Desktop treatment	Mobile treatment (≤ 640 px)
Global header	full-width bar	keep height ≤ 56 px; collapse rarely-used links into a “⋮” menu
Primary KPI cards (Total Committed, Annualized Return, Active Deals)	3-column grid	single horizontal swipeable carousel (snap-scroll) or a 3-row stack
“Portfolio Diversification Opportunity” banner	full-row banner with CTA buttons	reduce copy-length, move CTAs beneath text, allow horizontal scroll if copy wraps
Tabs (Portfolio / Opportunities / Docs)	inline pill tabs	stick them horizontally scrollable just below header; use overflow-x-auto
Side-panel mockup (new compact design)	sidebar icons	swap for a bottom nav bar with 4–5 icons (position: fixed; bottom: 0)
Quick Actions	row of 4 buttons	convert to vertical list or 2 × 2 grid; keep the same IDs so JS hooks don’t break
Table-like “Your Investments” rows	four numeric columns	two-line card:
line 1 → deal name + status badge
line 2 → small grid: Committed

Implementation tip: Tailwind’s responsive prefixes (sm:, md:, lg:) let you keep the existing classes and simply add mobile overrides.

2. Interaction polish ⚡
Concern	Mobile UX fix
Tap targets	ensure buttons & chips ≥ 44 × 44 px
Scrolling fatigue	Collapsible “Portfolio Health”, “Your Investments”, etc. via <details>/<summary> or a custom accordion
Sticky feedback	Use Toast notifications (toast.bottom-center) instead of banners that push content down
Data-dense tables	Offer a “Compact / Expanded” toggle; compact mode hides less-used columns behind a “…” chevron
API latency (field use)	skeleton loaders & optimistic UI; cache last-used property locally so results show instantly offline

4. Performance & field reliability 📶
Install Service Worker → cache CSS/JS shell & last valuation call for offline viewing.

Lazy-load images with loading="lazy" + a blurred placeholder.

Compress JS bundle with esbuild --target=es2019.

5. QA checklist ✅
iPhone SE / Pixel 4 – no horizontal scroll.

Tap any KPI card → ripple feedback within 100 ms.

Rotate phone → grid re-flows, no overlap.

Offline mode (DevTools ⇢ Network ⇢ Offline) still shows last finished valuation page.

200 ms throttled network → skeletons appear, toast on API errors.

Hand this spec to Replit / Ghostwriter and ask them to:

Add the Tailwind mobile classes & bottom nav bar.

Keep existing IDs, hooks, and Acquisitions / Dispositions JS untouched.

Verify the responsive breakpoints with Storybook or Chrome Device Mode.

Once merged, you’ll have a tight, elegant mobile dashboard your teams can trust in the field.
